# Plan: v0.3 — Email Caching

## Context

`gmail_summarize_school_emails` fetches multiple emails and returns them as one concatenated string. For 5 emails this hit ~112K characters, exceeding MCP tool result limits. The fix: save emails as local markdown files on fetch, then return file paths instead of bodies. Claude reads the files directly (no size limit). Emails never change once sent, so caching is safe with no staleness concern.

File format — YAML frontmatter + body (Obsidian-compatible):
```markdown
---
id: 19abc123def
subject: Weekly Newsletter
from: office@school.edu
date: Mon, 1 Apr 2026
---

(email body as markdown)
```

Files organized by year/month from the email's date header:
```
emails/
├── 2026/
│   ├── 03/
│   │   └── 19abc123def.md
│   └── 04/
│       └── 19def456abc.md
```

Parse date from email header via `email.utils.parsedate_to_datetime()`. Fallback to current year/month if date header is missing or unparseable.

No auto-recycling — emails are tiny files cached forever. Manual cleanup via `rm -rf emails/` if needed.

Reuses existing helpers: `get_header()`, `extract_body()`, `body_to_markdown()` from `gmail_helpers.py`.

Creates subdirectories on write via `path.parent.mkdir(parents=True, exist_ok=True)`.

## Changes to existing files

### `gmail_school.py`
Refactor `fetch_school_emails_with_context()` → `prepare_school_summary()`:
- Call `ensure_cached(service, message_ids)` to get file paths
- Return: personal context + summary instructions + list of absolute file paths
- Do NOT include email bodies in return value
- `_format_personal_context()` and `_load_summary_instructions()` unchanged

### `gmail_api.py`
Wire `fetch_body()` through cache:
- Check `find_cached(message_id)` → return `read_cached(message_id)` if hit
- Otherwise call `save_email(service, message_id)` then return `read_cached(message_id)`
- Same return type and signature

### `server.py`

- `gmail_fetch_body` unchanged (caching is transparent inside `gmail_api.py`)

## New workflow

1. Claude calls `gmail_list_messages("from:synapseschool.org is:unread")` → gets IDs + subjects
2. Claude calls `gmail_summarize_school_emails(["id1", "id2"])` → gets back ~500 chars: personal context, instructions, file paths like `/Users/anna/src/gmail-assistant/emails/2026/04/19abc123.md`
3. Claude reads each file directly (no MCP size limit)
4. Claude writes the summary
