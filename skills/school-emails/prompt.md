Structure the summary in this exact order:

## Action Items
Things I need to do, with deadline in bold. Each action item that has a deadline MUST also appear in the Calendar section below.

## Calendar
A single chronological list of relevant dates and events — this is the canonical source of truth for dates. No other section may introduce a new date. Group by date, oldest first. Tag child-specific entries inline with [Child Name]. Include only events that involve one of my children, apply to all families (closures, parent events, school-wide deadlines), or require me to act. Skip generic school sports, performances, or activities that don't involve my children.

## Per-Child Notes
For each child mentioned in the emails, a short section with non-date context only: classroom themes, ongoing topics, teacher shoutouts, behavioral notes, etc. Do NOT repeat dates here — reference events by name and let the reader cross-reference the Calendar.

## Emails Included
Markdown table of the emails parsed with columns: Date | Sender | Subject. One row per email, sorted by date.

Style rules:
- Keep it concise — bullet points, not prose.
- The school uses multi-age "levels" like 5/6 that span two traditional grades; emails may reference either grade or level.
- Every date must appear in the Calendar section exactly once. If you find yourself wanting to write a date inside Action Items or Per-Child Notes, instead reference the event by name and ensure it's in the Calendar.

Save the summary to a file in your current directory: school-summary-YYYY-MM-DD.md
